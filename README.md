## 🤖 Machine Learning

### Algorithm: Random Forest Classifier

- **Why RF?** Handles non-linear relationships, resistant to overfitting, provides feature importances, no scaling required for tree-based splits
- **Hyperparameter tuning:** GridSearchCV with 5-fold cross-validation
- **Class weighting:** `class_weight='balanced'` to handle imbalanced classes
- **Why RF over Logistic Regression?** LR achieved marginally higher raw accuracy (74.57% vs 73.83%), but Random Forest was selected as the best model based on **CV AUC (0.827 vs 0.833)** — a better indicator of real-world generalization. RF also provides native feature importances and superior interpretability for this use case.

### Best Model Hyperparameters

| n_estimators | 200 |
| max_depth | 20 |

## 📊 Dataset (15,000 records)

### Features (35 total)

**Raw features (13):**
- `airline`, `dep_airport`, `arr_airport`, `aircraft_type` (categorical)
- `month`, `day_of_week`, `dep_hour`, `weather_severity`, `wind_speed`, `visibility`, `congestion`, `distance`, `prev_delay` (numeric)

**Engineered features (22):**
- Time flags: `is_weekend`, `is_rush_hour`, `is_night`, `is_summer`, `is_holiday`, `is_spring_break`
- Weather flags: `is_severe_wx`, `is_moderate_wx`, `is_low_vis`, `is_high_wind`, `is_storm`
- Interactions: `wx_wind`, `wx_vis`, `cong_rush`, `cong_weekend`
- Composites: `risk_score`, `weather_sq`, `wind_sq`, `vis_inv`, `log_dist`, `log_prev_delay`, `prev_delay_flag`

**Target:** `is_delayed` — 1 if delay > 15 min, 0 otherwise

### Prediction Input (UI Form)

The dashboard prediction form accepts these user-facing inputs:

| Field | Type | Notes |
|---|---|---|
| Departure Airport | Dropdown | JFK, LAX, ORD, DFW, DEN, ATL, SFO, SEA, MIA, BOS |
| Arrival Airport | Dropdown | Same options |
| Flight Date | Date picker | Used to derive month + day of week |
| Departure Time | Time picker | Used to derive dep_hour |
| Airline | Dropdown | AA, DL, UA, WN, B6, AS, NK |
| Aircraft Type | Dropdown | 737, A320, 777, 787 |
| Weather Severity | Slider 1–10 | — |
| Airport Congestion | Slider 1–10 | — |
| Wind Speed (knots) | Number input | — |
| Visibility (miles) | Number input | — |

## 🔌 API Endpoints

| Method | Endpoint | Description |
|---|---|---|
| GET | `/` | Web UI (serves index.html) |
| POST | `/predict` | Get delay prediction from Random Forest |
| GET | `/model-info` | RF metrics + hyperparams from model_meta.pkl |
| GET | `/stats` | Dataset + DB statistics |
| GET | `/predictions/history` | Last N predictions from DB |
| GET | `/db/summary` | Delay stats by airline/airport |
| GET | `/plots/<name>` | Serve evaluation chart images |
| GET | `/health` | Server status |

### Example: POST /predict

**Request:**
```json
{
  "dep_airport": "ORD",
  "arr_airport": "JFK",
  "airline": "AA",
  "aircraft_type": "737",
  "dep_hour": 17,
  "month": 7,
  "day_of_week": 4,
  "weather_severity": 7,
  "congestion": 8,
  "wind_speed": 35,
  "visibility": 4,
  "distance": 740,
  "prev_delay": 20
}
```

**Response:**
```json
{
  "prediction": 1,
  "status": "SIGNIFICANT DELAY",
  "delay_minutes": 41,
  "confidence": 84.2,
  "probability": 0.842,
  "model_used": "Random Forest",
  "ml_powered": true,
  "factors": {
    "weather": 33,
    "congestion": 28,
    "carrier": 18,
    "time_of_day": 21
  }
}
```

> **Fallback:** If the Flask server is unreachable, the frontend applies a rule-based calculation using weather severity, congestion, wind speed, visibility, airline bias, and time-of-day factors — clearly labelled as "Rule-Based Fallback" in the UI.
